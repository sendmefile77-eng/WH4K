# Myauko RPG Engine (WH4K Modular)

**Модульный движок взрослой grimdark эротической RPG** с динамической сборкой промптов и подключаемыми модулями.

## Концепция

- **Ядро (Core Engine)** — неизменяемая основа игры: акты, кадры, State Vector, Body DNA, Erotic Director, умная сборка промптов, интеграция с API изображений.
- **Модули** — полностью заменяемый контент (банки, стили, сценарии, визуальная эстетика). Один активный модуль за раз.
- Приложение само собирает промпты под текущую ситуацию + данные модуля + команды игрока.
- Возможность кастомизации сути кадров в настройках с умной перестройкой промпта.
- Динамический хаб + адаптивная Мяуко (внешность зависит от тематики последнего посещения).

## Текущая архитектура (v0.1)

### 1. Core Engine
- **Game Loop**: Хаб → Выбор направления → Showcase → Visit (Act I–V с кадрами и паузами)
- **State System**:
  - `StateVector` (18 параметров)
  - `BodyDNA` (уникальное тело NPC)
  - `VisitIdentitySnapshot`
- **Prompt System**:
  - Умный `PromptAssembler`
  - Поддержка команд игрока ("больше фут-фокуса", "сделай это между сисек" и т.д.)
  - Умная перестройка промпта при кастомизации кадров
- **Image Integration**: Чистый API (планируется Atlas Cloud)

### 2. Module System
- Модуль = папка с `module.json` + `banks/`, `menus/`, `prompts/`, `styles/`
- Загрузка модулей вручную
- Хранение в приложении
- Переключение в настройках (только один активный)

### 3. Главный экран (Хаб)
- Динамический фон + атмосфера
- Мяуко меняет внешность под тематику последнего модуля
- Кнопки направлений из активного модуля

### 4. Визит и кадры
- Жёсткая структура актов
- Возможность менять суть каждого кадра в настройках
- Регенерация кадра во время визита по команде

## Технологии

- **Язык**: Kotlin
- **UI**: Jetpack Compose + Material 3
- **State**: Room + DataStore + ViewModel
- **Сетевое**: Ktor / Retrofit
