# Content Boundary

WH4K separates engine code from module content.

## Engine responsibilities

- State management.
- Visit loop.
- Prompt assembly.
- Module validation.
- Atlas Cloud integration.
- Persistence and settings.

## Module responsibilities

- Fictional world data.
- Directions and menus.
- Character/location/atmosphere banks.
- Visual style tags.
- Frame templates.

## Non-negotiable module validation

- Characters must be fictional adults.
- Minor-coded or underage content is rejected.
- Real-person likeness is rejected by default.
- Illegal content is rejected.
- Provider-specific safety rules are enforced at the Atlas boundary.

This document exists so the engine can be extended without weakening the validation boundary.
